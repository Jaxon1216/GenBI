## 核心概念

### 分页的基本原理

分页是将大量数据分成多个页面显示的技术，主要涉及以下参数：

- **current**: 当前页码（从 1 开始）
- **pageSize**: 每页显示的数据条数
- **total**: 数据总数

```
示例：
total = 100 条数据
pageSize = 10 条/页
总页数 = Math.ceil(100 / 10) = 10 页
```

### 后端分页 vs 前端分页

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **后端分页** | 性能好，减少数据传输 | 需要后端支持 | 大数据量（推荐） |
| **前端分页** | 实现简单，无需后端改动 | 一次性加载所有数据 | 小数据量 |

本项目使用的是 **后端分页**。

## 状态管理

### 1. 定义查询参数状态

```typescript
// 初始化搜索参数
const initSearchParams = {
  current: 1,        // 第一页
  pageSize: 4,       // 每页4条
};

// 使用 useState 管理查询参数
const [searchParams, setSearchParams] = useState<API.ChartQueryRequest>({
  ...initSearchParams,
});
```

**关键点**：
- `current` 和 `pageSize` 是分页的核心参数
- 使用对象展开运算符 `...` 确保创建新对象

### 2. 定义数据状态

```typescript
const [chartList, setChartList] = useState<API.Chart[]>([]);  // 当前页数据
const [total, setTotal] = useState<number>(0);                // 数据总数
const [loading, setLoading] = useState<boolean>(false);       // 加载状态
```

**状态说明**：
- `chartList`: 存储当前页的数据列表
- `total`: 存储数据库中符合条件的总记录数
- `loading`: 控制加载动画的显示/隐藏

## 数据加载

### 1. 加载数据函数

```typescript
const loadData = async () => {
  setLoading(true);  // 1️⃣ 开始加载，显示 loading
  
  try {
    // 2️⃣ 调用后端 API，传入查询参数
    const res = await listMyChartByPageUsingPost(searchParams);

    if (res.data) {
      // 3️⃣ 更新列表数据和总数
      setChartList(res.data.records ?? []);
      setTotal(res.data.total ?? 0);
    } else {
      message.error('获取我的图表失败');
    }
  } catch (e: any) {
    message.error('获取我的图表失败,' + e.message);
  } finally {
    setLoading(false);  // 4️⃣ 加载完成，隐藏 loading
  }
};
```

**执行流程**：
```
用户操作 → 修改 searchParams → 触发 loadData 
         → 后端返回数据 → 更新 state → 页面重新渲染
```

### 2. 自动触发加载

```typescript
useEffect(() => {
  loadData();
}, [searchParams]);  // 依赖 searchParams
```

**工作原理**：
- 当 `searchParams` 发生变化时，自动执行 `loadData()`
- 首次渲染时也会执行一次
- 实现自动加载和分页切换

## 分页处理

### 1. 分页组件配置

```typescript
<Pagination
  current={searchParams.current}           // 当前页码
  pageSize={searchParams.pageSize}         // 每页条数
  total={total}                            // 数据总数
  showTotal={(total) => `共 ${total} 条`}  // 显示总数
  showSizeChanger                          // 显示页码切换器
  pageSizeOptions={['4', '8', '12', '16']} // 可选的每页条数
  onChange={handlePageChange}              // 页码或每页条数改变时的回调
/>
```

### 2. 分页切换处理函数

```typescript
const handlePageChange = (page: number, pageSize: number) => {
  setSearchParams({
    ...searchParams,  // 保留其他查询条件
    current: page,    // 更新页码
    pageSize,         // 更新每页条数
  });
};
```

**触发流程**：
```
用户点击页码 → handlePageChange 被调用 
            → setSearchParams 更新状态 
            → useEffect 监听到变化 
            → loadData 重新加载数据
```

## 列表渲染

### 1. 响应式网格布局

```typescript
<Row gutter={[16, 16]}>  {/* 水平和垂直间距都是 16px */}
  {chartList.map((chart) => (
    <Col 
      xs={24}   // 超小屏幕（<576px）：1列
      sm={24}   // 小屏幕（≥576px）：1列
      md={12}   // 中等屏幕（≥768px）：2列
      lg={12}   // 大屏幕（≥992px）：2列
      xl={12}   // 超大屏幕（≥1200px）：2列
      key={chart.id}
    >
      {/* 卡片内容 */}
    </Col>
  ))}
</Row>
```

**栅格系统说明**：
- Ant Design 使用 24 栅格系统
- `xs={24}` 表示占满整行（24/24 = 100%）
- `md={12}` 表示占半行（12/24 = 50%），即 2 列布局

### 2. 卡片内容结构

```typescript
<Card
  className="chart-card"
  hoverable                    // 鼠标悬停效果
  title={
    <Space>
      <span>{chart.name || '未命名图表'}</span>
      {chart.chartType && <Tag color="blue">{chart.chartType}</Tag>}
    </Space>
  }
>
  {/* 1. 分析目标 */}
  <Paragraph ellipsis={{ rows: 2 }} type="secondary">
    <strong>分析目标：</strong>
    {chart.goal || '无'}
  </Paragraph>

  {/* 2. ECharts 图表 */}
  {chartOption && (
    <ReactECharts 
      option={chartOption} 
      style={{ height: '300px' }} 
    />
  )}

  {/* 3. 分析结论 */}
  {chart.genResult && (
    <Paragraph ellipsis={{ rows: 3, expandable: true }}>
      <strong>分析结论：</strong>
      {chart.genResult}
    </Paragraph>
  )}
</Card>
```

### 3. 图表数据解析

```typescript
// 后端返回的 genChart 是 JSON 字符串
const chartOption = chart.genChart ? JSON.parse(chart.genChart) : null;

// 示例数据：
// chart.genChart = '{"title":{"text":"用户增长"},"xAxis":{...},"yAxis":{...}}'
// 解析后成为 ECharts 配置对象
```

└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐

└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐

### 关键技术点总结

#### 1. **状态驱动视图**
```typescript
searchParams 变化 → 触发数据加载 → 更新视图
```

#### 2. **useEffect 依赖数组**
```typescript
useEffect(() => {
  loadData();
}, [searchParams]);  // ⚠️ 只监听 searchParams
```

**注意**：不要将 `loadData` 加入依赖数组，会导致无限循环！

#### 3. **对象的不可变更新**
```typescript
// ❌ 错误：直接修改对象
searchParams.current = 2;
setSearchParams(searchParams);  // React 不会检测到变化

// ✅ 正确：创建新对象
setSearchParams({
  ...searchParams,
  current: 2,
});
```

#### 4. **条件渲染**
```typescript
{chartList.length === 0 && !loading ? (
  <Result title="暂无图表" />  // 空状态
) : (
  <Row>...</Row>                // 数据列表
)}
```

## 🐛 常见问题

### Q1: 分页不工作，数据不刷新？

**原因**：`searchParams` 对象引用未改变

**解决**：使用展开运算符创建新对象
```typescript
setSearchParams({ ...searchParams, current: page });
```

### Q2: 页面一直 loading？

**原因**：忘记在 `finally` 中设置 `setLoading(false)`

**解决**：
```typescript
try {
  // ...
} finally {
  setLoading(false);  // 确保无论成功失败都会执行
}
```

### Q3: useEffect 无限循环？

**原因**：依赖数组中包含了会改变的函数或对象

**解决**：
```typescript
// ❌ 错误
useEffect(() => {
  loadData();
}, [searchParams, loadData]);  // loadData 每次渲染都是新函数

// ✅ 正确
useEffect(() => {
  loadData();
}, [searchParams]);  // 只依赖 searchParams
```

### Q4: 切换页码后滚动条不回到顶部？

**解决**：在分页切换时滚动到顶部
```typescript
const handlePageChange = (page, pageSize) => {
  setSearchParams({ ...searchParams, current: page, pageSize });
  window.scrollTo({ top: 0, behavior: 'smooth' });  // 平滑滚动到顶部
};
```
